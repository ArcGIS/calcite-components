#Issue1: No height & width set on flex container. height and width are set with calcite-text-area selector.
When the resizing start observe that both calcite-text-area and native textarea pushes the element's around in the layout. This behavior is consistent with native textarea. To achieve such behavior in calcite-text-area we are setting the height and width of host to auto when resize begins. If not, when resizing starts overlapping is observed.

#Issue2: Height & width are set on both flex container and calcite-text-area selector.
When the resizing start , calcite-text-area overlaps with the neighboring elements in the layout where as native textarea does overlap but restrict the user to not resize beyond the flex container height. This is due to flex-direction being column. In calcite-texta-rea case this is not happening due to the height, width being set to auto as mentioned in above issue 1.
